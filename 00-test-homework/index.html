git config --global user.name - задає глобальне ім'я для всіх наступних комітів git config --global user.email - задає глобальний мейл для всіх наступних комітів git init - створює та/або ініціалізує вже створену папку для гіта. git status - показує стан файлів у робочому каталозі та індексі git add - додає вміст робочого каталогу в індекс для наступного коммита git commit - бере всі дані, додані до індексу за допомогою git add, і зберігає їх зліпок у внутрішній базі даних, а потім переносить вказівник поточної гілки на цей зліпок. git log - використовується для перегляду історії коммітів git checkout - використовується для перемикання гілок та вивантаження їхнього вмісту в робочий каталог. git branch - це "менеджер гілок". git push - використовується для встановлення зв’язку з видаленим репозиторієм, вирахування локальних змін і їх передачі у цей репозиторій. git pull - спочатку збирає зміни із зазначеного видаленого репозиторію, а потім намагається злити їх з поточною гілкою. git merge - використовується для злиття однієї або кількох гілок у поточну і робить її активною. git clone - створення клону гіт-репозитарію на лок. пк .gitignore - файли та папки, котрі ігноруються гітом git revert - створює новий коміт, який вносить зміни, протилежні до зазначеного комміту, по суті скасовуючи його. git reset - скасовує зміни git rebase - виконує таке ж, як cherry-pick, але для ланцюжка коммітів, тим самим ніби переносячи гілку на нове місце. git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці. git stash - використовується для тимчасового збереження всіх незафіксованих змін для очищення робочого каталогу без необхідності фіксувати незавершену роботу в поточній гілці.
